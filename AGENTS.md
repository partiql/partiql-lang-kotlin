# agents.md

This file provides guidance to AI coding agents when working with code in this repository.

## Project Overview

PartiQL Lang Kotlin is a Kotlin/JVM implementation of the PartiQL specification — a SQL-compatible query language extended for schemaless hierarchical data. The project is published to Maven Central under `org.partiql`.

## Build Commands

**Requires Java 11+. Uses Gradle with parallel builds and caching.**

```bash
# Initialize submodules (required for conformance tests)
git submodule update --init --recursive

# Build everything
./gradlew assemble

# Run all tests (includes ktlint check)
./gradlew build

# Run tests for a specific subproject
./gradlew :partiql-eval:test
./gradlew :partiql-parser:test
./gradlew :partiql-planner:test

# Run a single test class
./gradlew :partiql-eval:test --tests "org.partiql.eval.internal.PartiQLEngineDefaultTest"

# Lint
./gradlew ktlintCheck
./gradlew ktlintFormat

# Run custom ktlint rules (not enabled by default)
./gradlew ktlintCheck -Pcustom-ktlint-rules

# Run conformance tests
./gradlew :test:partiql-tests-runner:test

# Run randomized tests
./gradlew :test:partiql-randomized-tests:test -PrandomTests

# Generate planner internal IR (after modifying partiql_plan_internal.ion)
./gradlew :partiql-planner:generate

# Install CLI locally
./gradlew :partiql-cli:install
# Then run: partiql-cli/build/install/partiql/bin/partiql
```

## Architecture

The compilation pipeline flows: **Parser → AST → Planner → Plan → Eval**

### Subproject Dependency Graph

```
partiql-lang (umbrella — depends on all below)
├── partiql-parser    → partiql-ast, partiql-spi
├── partiql-planner   → partiql-plan, partiql-ast, partiql-spi
├── partiql-eval      → partiql-ast, partiql-plan, partiql-planner, partiql-spi
└── partiql-cli       → all of the above (application entry point)
```

- **partiql-spi** — Foundation layer. Types (`PType`), values (`Datum`), catalogs, connectors, and all built-in function implementations (`spi/function/builtins/`). Both Java and Kotlin sources.
- **partiql-ast** — AST node classes (Java with Lombok). Visitor pattern via `AstVisitor`. SQL serialization in `ast/sql/`.
- **partiql-parser** — ANTLR4 grammar (`src/main/antlr/PartiQLParser.g4`, `PartiQLTokens.g4`) → AST. The ANTLR runtime is shadowed/relocated to avoid classpath conflicts.
- **partiql-plan** — Logical plan IR nodes (Java with Lombok). `rel/` for relational operators, `rex/` for row expressions.
- **partiql-planner** — AST → Plan. Type inference (`internal/typer/`), cast resolution (`internal/casts/`), transforms (`internal/transforms/`). Has its own internal IR generated by Sprout from `src/main/resources/partiql_plan_internal.ion`.
- **partiql-eval** — Plan → execution. Physical operators in `internal/operator/rel/` and `internal/operator/rex/`.
- **partiql-cli** — PicoCLI application. Shell, query execution, format command.
- **lib/sprout** — Code generator for IR node classes from Ion type definitions.

### Key Conventions

- **Explicit API mode** is enforced (strict) on library subprojects — all public declarations need explicit visibility modifiers and return types.
- **Internal code** lives in `impl` or `internal` packages.
- **Java and Kotlin coexist** — public API classes are increasingly written in Java (with Lombok); internal implementation stays in Kotlin.
- **JVM target is 1.8** for published libraries (source compiled with Kotlin 1.9).
- Tests use **JUnit 5** (JUnit Platform). Some older tests use JUnit 4 via the Vintage engine.
- ktlint check runs automatically before tests (`tasks.test` depends on `tasks.ktlintCheck`).
